## Directory Purposes

**configs/**
- Purpose: Application configs.
- Contains: `dataset.yaml`, `taxonomy.yaml`, `llm.yaml`, `asr.yaml`, `report.yaml`.
- Key files: `dataset.yaml` for HF credentials and metadata schema fields.

**data/**
- Purpose: Directory tree to house data artifacts.
- Subdirectories:
  - `raw/vimedcss/ViMedCSS-Metadata/`: Stores downloaded CSV metadata files (`train_set.csv`, `valid_set.csv`, `test_set.csv`, `hard_set.csv`).
  - `interim/`: Cache or temp tables.
  - `processed/`: Standardized, clean datasets.

**src/**
- Purpose: Core application modules and CLI.
- Contains: `cli.py` (CLI entry point).
- Subdirectories:
  - `hf_client/`: Downloader logic using `huggingface_hub`.
  - `audit/`: Schema mapper and statistical verification.
  - `shared/`: AppConfig loader and logger configurations.
  - `terms/`, `llm/`, `asr/`, `metrics/`, `reports/`: Empty directory structures ready for pipeline implementation.

**outputs/**
- Purpose: Exported files generated by running pipeline commands.
- Subdirectories:
  - `audit/`: CSV statistical logs, manifest metadata, `metadata_schema_report.md`.
  - `term_coverage/`, `asr_eval/`, `reports/`: Target locations for subsequent phase deliverables.

**tests/**
- Purpose: Automated unit tests directory.
- Contains: `test_*.py` files testing configurations, download clients, and auditors.

## Key File Locations

**Entry Points:**
- `src/cli.py` - Main CLI driver.
- `Makefile` - Terminal automation.

**Configuration:**
- `configs/dataset.yaml` - Input datasets and columns mapping parameters.
- `configs/taxonomy.yaml` - Custom term classification frequency boundaries.

**Testing:**
- `tests/test_auditor.py` - Auditor class unit tests.
- `tests/test_config.py` - Config loading tests.
- `tests/test_hf_client.py` - Download client tests.

## Naming Conventions

**Files:**
- `snake_case.py` for source files and test scripts.
- `snake_case.csv` or `snake_case.json` for output datasets/metadata.
- `kebab-case.yaml` for configuration files.
- `UPPERCASE.md` or `lowercase.md` for documents.

**Directories:**
- `snake_case` for codebase python module packages (`hf_client`, `term_coverage`, `asr_eval`).

## Where to Add New Code

**New Pipeline Phase (e.g. Term Extraction):**
- Implementation: `src/terms/`
- Command hook: Register subcommand inside `src/cli.py` and map a target in the `Makefile`.
- Tests: Add a new `tests/test_terms.py` script.

**New Evaluation Metric:**
- Implementation: `src/metrics/`
- Integration: Update testing assertions in `tests/test_metrics.py`.

**New Config Option:**
- YAML schema: Add file to `configs/`.
- Loader: Update `AppConfig` in `src/shared/config.py` to expose a getter.

## Special Directories

**data/raw/**
- Purpose: Temporary dataset caching.
- Source: Hugging Face dataset downloads.
- Committed: No (specifically excluded in `.gitignore`).

**outputs/**
- Purpose: Pipeline outputs.
- Source: Automatically written during pipeline runs.
- Committed: No (contents gitignored; directory structure kept using `.gitkeep` files).
